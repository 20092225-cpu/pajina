Mi Investigación sobre la Historia de HTML

Un repaso detallado desde sus orígenes en los laboratorios de física hasta la web moderna.

1. El Nacimiento del Lenguaje (1989 - 1991)

Descubrí que HTML nació gracias al científico Tim Berners-Lee, quien trabajaba en el CERN (Organización Europea para la Investigación Nuclear). Él necesitaba una forma de compartir documentos de investigación entre científicos de todo el mundo de manera rápida y eficiente.

En 1991, publicó el primer documento oficial llamado "HTML Tags", el cual contenía apenas 18 etiquetas básicas. Muchas de esas etiquetas primitivas, como los párrafos o los títulos, todavía las sigo utilizando en mis códigos actuales.

2. La Guerra de Navegadores y la Estandarización (1995 - 1999)

A medida que la web se hacía popular, empresas como Netscape y Microsoft empezaron a inventar sus propias etiquetas sin consultar a nadie. Esto causó un caos porque las páginas se veían bien en un navegador pero rotas en otro.

Para solucionar este problema, el W3C (World Wide Web Consortium) tomó el control del lenguaje y lanzó estas versiones clave:

  • HTML 2.0 (1995): El primer estándar oficial. Aquí ya se podían usar formularios básicos y tablas.
  • HTML 3.2 (1997): Añadió soporte para tablas más complejas, imágenes alineadas y applets de Java.
  • HTML 4.01 (1999): Una versión extremadamente estable que separó por primera vez el contenido (HTML) del diseño (CSS). Dominó la web por más de diez años.

3. El Fracaso de XHTML y el Éxito de HTML5 (2000 - Presente)

A principios de los años 2000, el W3C intentó cambiar HTML por XHTML, un lenguaje mucho más estricto que bloqueaba la pantalla del usuario si había un solo error de código. A los desarrolladores no les gustó este cambio tan radical.

Debido a esto, un grupo de empresas (Apple, Mozilla y Opera) fundó el WHATWG para crear un HTML más flexible y moderno. Este esfuerzo dio nacimiento a HTML5, lanzado oficialmente en 2014. HTML5 revolucionó el internet porque eliminó la necesidad de instalar complementos externos como Adobe Flash para ver videos o escuchar audio.

4. Lo que representa HTML hoy en día

Hoy entiendo que HTML ya no se actualiza por versiones congeladas (como HTML6), sino que se considera un "Living Standard" (Estándar Vivo). Esto significa que el lenguaje evoluciona y recibe mejoras continuas todos los meses para adaptarse a los nuevos dispositivos y necesidades del mundo digital.
